Opzioni di Deployment

Azure App Service supporta molteplici metodi di deployment per adattarsi a qualsiasi workflow, dal semplice upload FTP alle pipeline CI/CD complesse.

Comandi CLI per Deployment

# Creare un App Service Plan (Standard S1)
az appservice plan create \\
  --name MyPlan \\
  --resource-group MyRG \\
  --sku S1 \\
  --is-linux

# Creare una Web App
az webapp create \\
  --name MyUniqueApp \\
  --resource-group MyRG \\
  --plan MyPlan \\
  --runtime "NODE:18-lts"

# Deploy da file ZIP
az webapp deployment source config-zip \\
  --resource-group MyRG \\
  --name MyUniqueApp \\
  --src app.zip

Tips per l'Esame AZ-104
  • I Deployment Slots sono disponibili solo nei tier Standard, Premium e Isolated.
  • Lo slot di "staging" gira sullo stesso hardware (App Service Plan) della produzione -> condividono le risorse!
  • Durante lo Swap, le impostazioni "Slot sticky" (come connection string specifiche di staging) rimangono nello slot di origine.
  • Lo swap riscalda (warm-up) l'app nello slot sorgente prima di scambiare i DNS, garantendo Zero Downtime Deployment.
