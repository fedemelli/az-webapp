Operazioni Bulk in Microsoft Entra ID

Le operazioni bulk permettono di gestire un gran numero di utenti utilizzando file CSV. Sono ideali per onboarding massivo, cleanup account, e gestione su larga scala.

Ruolo Prerequisito

Ruolo minimo richiesto: User Administrator

Operazioni Disponibili

Bulk Create

Crea utenti da CSV. Ideale per onboarding massivo di nuovi dipendenti.

Bulk Invite

Invita guest users esterni tramite B2B. Invia email di invito in massa.

Bulk Delete

Elimina utenti in massa. Attenzione: irreversibile dopo 30gg.

Download Users

Esporta lista utenti in CSV per backup, audit o analisi.

Bulk Restore

Ripristina utenti eliminati (entro 30 giorni) in massa.

Formato CSV Obbligatorio

Il file CSV deve rispettare una struttura precisa. Scarica sempre il template dal portale per avere il formato corretto.

PowerShell per Automazione

Per scenari pi√π complessi o integrazione con pipeline CI/CD, usa PowerShell con Microsoft Graph:

# Connettersi a Microsoft Graph
Connect-MgGraph -Scopes "User.ReadWrite.All"

# Importare utenti da CSV
$users = Import-Csv -Path "users.csv"

foreach ($user in $users) {
    # Crea oggetto password
    $PasswordProfile = @{
        Password = $user.Password
        ForceChangePasswordNextSignIn = $true
    }

    # Crea utente
    New-MgUser -DisplayName $user.DisplayName `
        -UserPrincipalName $user.UPN `
        -PasswordProfile $PasswordProfile `
        -AccountEnabled $true `
        -MailNickname $user.MailNickname `
        -UsageLocation "IT"  # Per assegnare licenze!

    Write-Host "Created:" $user.UPN
}

# Verificare utenti creati
Get-MgUser -Filter "UserType eq 'Member'" | Select-Object DisplayName, UserPrincipalName

Azure CLI Alternative

# Loop su file CSV con bash
while IFS=, read -r name upn password; do
    az ad user create \
        --display-name "$name" \
        --user-principal-name "$upn" \
        --password "$password" \
        --force-change-password-next-sign-in true
done < users.csv
