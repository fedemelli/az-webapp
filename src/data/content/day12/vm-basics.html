Azure Virtual Machines

Le Azure Virtual Machines (VM) offrono risorse di calcolo scalabili e on-demand. Sono la base dell'IaaS (Infrastructure as a Service) in Azure.

Parametri di Configurazione

Size (Dimensione)
Definisce CPU, RAM e storage temporaneo. Es: Standard_D2s_v3
Image (Immagine)
Sistema operativo: Windows Server, Ubuntu, Red Hat, Debian, ecc.
Authentication
Password (Windows) o SSH public key (Linux raccomandata)
Networking
VNet, Subnet, NSG, Public IP, Load Balancer

Serie di VM Azure

Azure offre diverse serie di VM ottimizzate per casi d'uso specifici:

Serie Ottimizzata per Casi d'Uso
B-series Burstable CPU Dev/test, web server a basso traffico
D-series General Purpose Produzione generale, database, app enterprise
E-series Memory Optimized SAP HANA, database in-memory, caching
F-series Compute Optimized Batch processing, gaming server, analytics
L-series Storage Optimized Big data, SQL, NoSQL, data warehousing
N-series GPU Machine learning, rendering, simulazioni

Creare una VM con Azure CLI

# Creare una VM Linux con SSH key
az vm create \\
  --resource-group myResourceGroup \\
  --name myVM \\
  --image Ubuntu2204 \\
  --size Standard_D2s_v3 \\
  --admin-username azureuser \\
  --generate-ssh-keys \\
  --public-ip-sku Standard

# Creare una VM Windows
az vm create \\
  --resource-group myResourceGroup \\
  --name myWindowsVM \\
  --image Win2022Datacenter \\
  --size Standard_D2s_v3 \\
  --admin-username azureuser \\
  --admin-password "ComplexP@ssw0rd!"

Naming Convention per VM Size

Capire il nome della VM size: Standard_D4as_v5

Standard_D4as_v5
D = Family
4 = vCPUs
a = AMD CPU
s = Premium Storage
v5 = Version
