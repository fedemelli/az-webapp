Azure Load Balancer

Azure Load Balancer opera al Layer 4 (Transport Layer - TCP/UDP) del modello OSI. Distribuisce il traffico in entrata tra istanze di backend sane.

Tips per l'Esame AZ-104
  • Load Balancer lavora a Layer 4 e usa un meccanismo di distribuzione basato su Hash (5-tuple: Source IP, Source Port, Dest IP, Dest Port, Protocol). NON guarda il contenuto (URL, cookie).
  • Health Probes sono essenziali: se la probe fallisce per una VM, il LB smette di mandare traffico a QUELLA VM (non riavvia la VM).
  • Il traffico di ritorno dal backend NON passa necessariamente attraverso il LB (Direct Server Return / Floating IP), ma per l'esame ricorda il flusso standard.
