Azure Storage Account

Un Azure Storage Account fornisce uno spazio dei nomi univoco in Azure per i tuoi dati. È il contenitore di livello più alto per tutti i servizi di storage Azure.

Servizi di Storage

Tipi di Storage Account

Tipo Servizi Supportati Performance Uso Tipico
Standard GPv2 Blob, File, Queue, Table Standard (HDD) Uso generale (raccomandato)
Premium Block Blobs Solo Block Blobs Premium (SSD) Alta frequenza transazioni
Premium File Shares Solo Azure Files Premium (SSD) Enterprise file shares
Premium Page Blobs Solo Page Blobs Premium (SSD) Dischi VM non gestiti

Access Tiers (Solo Blob Storage)

I tier di accesso bilanciano costi di storage e costi di accesso:

  • Hot: Per dati ad accesso frequente. Costo storage più alto, costo accesso più basso.
  • Cool: Per dati ad accesso infrequente (almeno 30 giorni). Costo storage minore, costo accesso maggiore.
  • Cold: Per dati raramente acceduti (almeno 90 giorni). Ancora più economico per storage.
  • Archive: Per dati di archiviazione a lungo termine (almeno 180 giorni). Costo storage minimo, ma richiede rehydration (ore/giorni) per accedere.

Creare Storage Account con CLI

# Creare uno storage account Standard GPv2
az storage account create \\
  --name "mystorageaccount123" \\
  --resource-group "my-rg" \\
  --location "westeurope" \\
  --sku "Standard_LRS" \\
  --kind "StorageV2" \\
  --access-tier "Hot"

# Creare con tier Cool
az storage account create \\
  --name "myarchivest" \\
  --resource-group "my-rg" \\
  --location "westeurope" \\
  --sku "Standard_GRS" \\
  --access-tier "Cool"

Punti Chiave per l'Esame

  • Il tier Archive richiede rehydration prima dell'accesso
  • Premium accounts non supportano access tiers
  • Il tier può essere impostato a livello di account o blob
  • Il cambio tier ha costi early deletion se fatto prima del periodo minimo
