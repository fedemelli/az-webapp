Azure Storage Account

Lo Storage Account è l'unità fondamentale di storage in Azure, offrendo un namespace unico per i tuoi dati (https://<accountname>.blob.core.windows.net).

Tipi di Account

Account Kind Performance Descrizione
General Purpose v2 (GPv2) Standard (HDD/SSD) Lo standard attuale. Supporta Blob, File, Queue, Table, Access Tiers e Replication ZRS/GZRS. Usa questo nel 99% dei casi.
Premium Block Blobs Premium (NVMe SSD) Bassissima latenza. Ideale per database, analytics ad alte prestazioni. Non supporta Access Tiering.
Premium File Shares Premium (SSD) Progettato per carichi SMB/NFS intensivi (es. FSLogix profiles per Desktop Virtuali, SQL migration).

Tips per l'Esame
  • Archive Tier: I dati sono offline. Per leggerli devi fare "Rehydrate" verso Hot/Cool. Richiede ore (fino a 15h per Standard priority).
  • Early Deletion Fee: Se cancelli o sposti un blob da Cool prima di 30 giorni (o Archive prima di 180), paghi una penale pro-rata.
  • Gli account Premium non supportano tiering (sono sempre "Super Hot").
